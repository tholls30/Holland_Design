TARGET PharmaSolutions

Freelance contract to optimize and design internal application

Project overview

TARGET has redefined what’s possible in generating & analyzing real-world data and evidence.

The Problem

The main problem of this project was focused on finding ways to increase the efficiency and accuracy of data abstracted from digital and physical medical records. TARGET employs a number of people whose sole job is to focus on turning digital, pdf, and scanned records into organized and usable information in a database. The previously implemented process for ingesting this data was inefficient from multiple standpoints; starting with the organization of tasks a single employee would take on to the use of an outdated UI with various hacked workarounds that ultimately lead to a poor UX. Additionally, the QA process for determining data validity and accuracy was not optimized mostly due to the technological constraints of that same UI.

The Process

This project started off with a week on-site where I did a deep-dive with a cross-functional team to learn about the current process, how to use the data abstraction tools, and to observe the how employees were using them. I then interviewed the employees who were using the tools and obtained their thoughts on the process in place. I also interviewed the managers who oversaw the operation along with the tech team that was tasked with developing the new solution. I was able to learn a lot about the pain points of the abstraction process and how the company envisioned it moving forward into the next iteration. After several weeks of organizing the information collected and completing additional research the team had a hypothesis about how to improve the actual abstraction process and how the UI was slowing down the employees. With the help of the director of abstraction we developed a UX flow chart and identified the key pieces of the interface that we needed to improve. In order to facilitate these discussions I created user personas for each person that touched the project's scope and clearly outlined the goals, frustrations, and motivations of each role. After this, we created a test plan to trial the new proposed process in order to determine if our hypothesis was correct before designing an interface to support that same hypothesis.

The Conclusion

Unfortunately my contract was cut short before we started testing the new abstraction process or designing the UI that would support it. This was due to changes in leadership and the occurrence of the COVID-19 pandemic.

Additional Work

As I was waiting for information/input from stakeholders I worked on creating a design system for TARGET's internal tools that would also be used for the abstraction tool. In order to deliver a consistent product experience both internally and externally to the company I worked with their marketing department to make sure we were aligned on colors, fonts, icons etc. I used this design system to increase engineering and UX efficiency on another internal tool they were building for reporting. I also provided some guidance on product development practices and proper PM procedures.
